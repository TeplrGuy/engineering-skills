# Contributing

## Branch naming convention
- Format: `<type>/issue-<number>-<slug>`
- Allowed `type`: `feature`, `fix`, `docs`, `chore`
- Example: `docs/issue-3-contributing-workflow`

## Skill promotion channels (draft/candidate/stable)
- `draft`: active development only; never consumed by production repos.
- `candidate`: benchmarked and under review; limited validation usage.
- `stable`: approved for production sync across repositories.

## Required evidence for PRs
- Link the governing issue.
- Include deterministic validation output (commands run and pass/fail result).
- Confirm CI status is green before merge.
